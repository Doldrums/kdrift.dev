About

Kernel maintenance shouldn't be a manual process

We're building the infrastructure to keep Linux drivers aligned with kernel evolution — automatically, verifiably, continuously.

01 — Mission
Why we exist
Closing the gap between kernel evolution and the ecosystem that depends on it.

The Linux kernel evolves faster than the ecosystem around it can adapt. Every release introduces API changes that ripple through thousands of drivers. In-tree, maintainers coordinate these changes manually. Out-of-tree, teams are on their own.

KDRIFT started as a research project studying how LLMs handle kernel code migration — not just generating patches, but understanding why APIs change and validating that patches preserve correctness. The result is a system that treats migration as a closed-loop engineering problem, not a one-shot code generation task.

02 — Research
Grounded in science
KDRIFT is grounded in peer-reviewed research on LLM-assisted code migration, kernel API evolution, and automated program repair.
Research Area 01
Kernel API Evolution
Longitudinal analysis of internal API stability across 20 major kernel releases. Characterization of change types and their impact on driver code.
Research Area 02
LLM Code Migration
Evaluation of frontier and open-source models on kernel-specific migration tasks. DRIFTBENCH as the first reproducible benchmark for this domain.
Research Area 03
Automated Program Repair
Multi-agent architectures for iterative patch refinement with staged compiler and runtime feedback loops.

04 — Timeline
Milestones
From research prototype to production system.
Q1 2024

Research project begins. Initial prototype for in-tree API migration.

Q3 2024

DRIFTBENCH v1 released. 48 API migration cases with validation harness.

Q1 2025

Multi-agent architecture (LPA + GMA). 56% compile success rate.

Q3 2025

DRIFTBENCH expanded to 71 cases. KERN-7B training begins.

Q1 2026

Public beta launch. Enterprise pilot program opens.

Q2 2026

KERN-32B release. Out-of-tree driver support.
